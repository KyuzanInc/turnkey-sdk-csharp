# Test fixtures

Each subdirectory groups fixtures for one production file, plus a
provenance note explaining where the bytes came from.

## Provenance levels

- **upstream-test-vectors** — directly copied from the upstream
  `__tests__/` directory inside `codex-crypto-reviews/upstream-snapshots/`.
- **rfc** — standardized test vectors (RFC 5869, RFC 9180, etc.).
- **nist** — NIST CAVP P-256 vectors.
- **turnkey-sample** — sample bundles published by Turnkey in their
  developer docs (de-identified).
- **node-generated** — bytes generated by running the pinned upstream Node
  package inside `tests/Fixtures/Generators/`. The generator script,
  Node version, and output hash are recorded so reviewers can re-run.

## Layout

```
Fixtures/
├── encoding/
│   └── turnkey-encoding-vectors.json       — upstream-test-vectors
├── crypto/                                  — (added with Crypto.cs port)
├── api-key-stamper/                         — (added with ApiKeyStamper.cs port)
└── http/                                    — (added with Http.cs port)
```

## Re-generating

Fixtures sourced from upstream test files are stable as long as
[`codex-crypto-reviews/upstream-snapshots/`](../../codex-crypto-reviews/upstream-snapshots/)
is not bumped. If a pin is bumped, re-extract the upstream `__tests__/`
into the matching fixture file and update its sha256 in
`tarball-checksums.txt` next to the new tarball.
